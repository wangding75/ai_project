# Phase 0 行动方案：AI网文生死验证

> **目标**：用最低成本回答唯一问题——AI生成的网文，能否在番茄小说上获得真实留存？  
> **周期**：14天  
> **每日投入**：≤ 1小时  
> **总花费**：≤ 500元人民币  
> **成功标准**：3本书中至少1本，10章内书架比（加书架数/阅读数）> 5%

---

## 一、准备工作（第0天，约3小时，仅做一次）

### 1.1 账号注册

注册3个番茄作家助手账号，每个账号对应一本书。

**要求：**
- 3个账号使用不同手机号
- 不同设备或清除缓存后注册，账号之间不要有关联
- 全部完成实名认证（未实名作品不展示给任何人）
- 每个账号准备一张封面图（淘宝搜"小说封面定制"，5-10元/张，告知题材即可）

### 1.2 选题

根据番茄平台调研，选择以下3个方向，覆盖男频主流题材：

| 书号 | 题材 | 核心标签 | 选题理由 |
|---|---|---|---|
| 书A | 都市系统流 | 系统觉醒+逆袭+打脸 | 番茄推荐榜常驻，AI最容易写，套路高度标准化 |
| 书B | 末世空间 | 末世+空间囤积+基地建设 | 近期热度持续，情节模块化，读者需求确定 |
| 书C | 都市高武 | 隐藏高手+身份反转+战力碾压 | 平台受众广，《斩神》验证了题材天花板 |

**选题红线：不选爱情主线、不选女频、不选需要深度历史知识的题材。**
AI在纯逻辑爽感上表现相对稳定，在情感细腻度和历史考据上容易穿帮。

### 1.3 书名与简介

书名和简介决定点击率，是流量第一关。

**书名公式：** `[开局/觉醒/重生] + [核心金手指] + [身份/结果]`

建议书名：
- 书A：《开局觉醒SSS级系统，我无敌了》
- 书B：《末世降临：我的空间大了亿点点》
- 书C：《都市最强隐士》

**简介公式：**
```
第一句：点明主角起点（越低越好，制造落差）
第二句：点明金手指/核心能力
第三句：点明爽点方向（谁要倒霉）
最后一句：制造悬念或高光时刻
```

示例（书A）：
```
陈浩，一个被所有人看不起的废物赘婿，
觉醒了全球唯一的SSS级签到系统。
曾经嘲笑他的人，一个个跪倒在他脚下。
"你们让我滚？好，我滚了，但我会回来的。"
```

## 二、每日执行流程（第1-14天）

### 标准日程（约45分钟）

```
09:00  生成3本书各1章正文（约20分钟）
09:20  润色检查（约10分钟）
09:30  发布到番茄（约5分钟）
09:35  记录昨日数据（约10分钟）
```

### 润色红线（必须严格遵守）

这是验证有效性的关键约束。一旦开始改情节，验证的就不是AI能力，结论会失真。

**✅ 允许修改：**
- 错别字、标点错误
- 明显重复的词语（"他感到感到很痛"）
- 过于书面化的表达（"此刻"→"这时"）
- AI腔套话（"这让他陷入了深深的沉思"→直接删）

**❌ 禁止修改：**
- 情节走向和人物决策
- 段落结构和场景顺序
- 爽点设计和冲突设计
- 对话内容（除非有明显硬伤）

> 如果某章实在看不下去，记录原因，继续发布，不修改。这是数据，不是失败。

### Prompt 1：世界观与人物设定生成（每本书只用一次）

**用途：** 第0天生成，后续所有章节生成时作为固定输入。生成后保存为文本文件。

#### 书A 设定Prompt：都市系统流

```
请为一部番茄小说男频都市系统流小说生成完整设定文档。

【主角设定】
姓名：陈浩，25岁，男
起点：被岳父家看不起的上门女婿，公司底层员工，月薪3000
核心能力：SSS级签到系统（在特定地点签到可获得技能、属性、物品）
性格：表面隐忍低调，内心清醒果断，不主动惹事但绝不忍让
核心幻想：读者通过他实现"被所有人低估后的彻底碾压"

【系统规则——必须精确，后续章节严格遵守】
签到地点：公司、家、城市地标等现实场所
签到奖励类型：技能书、属性点、物品、金钱、人脉
等级体系：F/E/D/C/B/A/S/SS/SSS，陈浩当前F级
限制：每天最多签到3次，奖励随机但与地点相关
已获得的能力不可遗忘，不可被剥夺

【反派与配角】
岳父林国华：势利眼商人，最看不起陈浩，口头禅"废物"
妻子林雪：善良但懦弱，夹在中间
小舅子林浩：嚣张跋扈，喜欢当众羞辱陈浩
贵人：公司清洁阿姨张大妈（隐藏身份：某集团董事长母亲，第8章后揭露）

【禁止项——后续所有章节不得出现】
主角不能主动跪舔任何人
主角不能因感情软化放弃报复机会
系统奖励不能出现逻辑矛盾
反派不能无缘无故变好人
不能出现超出系统设定的能力

【第一章目标爽点】
林浩在公司当众羞辱陈浩，陈浩首次签到获得"洞察"技能，当场看穿林浩的秘密，用一句话反将一军，让林浩当场哑口无言。读者看完第一章就想看第二章。

输出：结构化设定文档，约600-800字，语言简洁，供后续章节生成直接引用。
```

#### 书B 设定Prompt：末世空间

```
请为一部番茄小说男频末世文生成完整设定文档。

【世界观】
末世类型：病毒爆发，人类变异为丧尸，同时部分人觉醒异能
时间节点：故事从末世第零天（病毒爆发当天凌晨）开始
地点：国内某二线城市

【主角设定】
姓名：李明，28岁，男
末世前职业：大型超市仓库管理员（这个身份很关键，他比任何人都知道物资在哪里）
核心能力：末世前一天意外激活异空间——独立存储空间，初始100平米
性格：务实冷静、有点社恐、不喜欢抱团，但讲义气，绝不坑队友
核心幻想：在乱世中掌控资源，游刃有余，不依赖任何人

【异空间规则——必须精确，后续严格遵守】
初始大小：100平米，可通过特定条件升级
时间静止：放入空间的食物不会腐烂，时间停止
存取方式：意念控制，旁人不可见，不会被发现
限制：每次存取有1分钟冷却时间；不能存入活的高等丧尸

【前5章必须完成的资源囤积清单】
食物：方便面500箱、矿泉水300箱、罐头200箱
医疗：常用药品、绷带、消毒用品若干
武器：钢管×5、厨刀×3（末世初期，枪支不合理）
特殊：蔬菜种子一批（为空间种植埋伏笔）
资源数量前后必须一致，送出去的必须减少

【配角设定】
前主管刘强：末世前就经常克扣工资欺负李明，末世后更嚣张，最终被打脸
邻居王大妈：照顾过李明，是他的第一个同伴，善良务实
觉醒者张伟：火系异能，傲慢，瞧不起没有异能的李明（不知道他有空间）

【禁止项】
主角不能轻易透露空间秘密（除非对已建立深度信任的人）
物资数量必须前后一致
不能出现没有来源的物资

【第一章目标爽点】
末世第零天，全城恐慌抢购，超市门口大打出手。李明一个人在仓库用异空间悄悄把整个仓库搬空，然后若无其事离开。当其他人在外面争抢最后几包泡面时，李明已经在自己家里喝着矿泉水，面前是装满物资的空间入口。

输出：结构化设定文档，约600-800字，语言简洁，供后续章节生成直接引用。
```

#### 书C 设定Prompt：都市高武

```
请为一部番茄小说男频都市高武文生成完整设定文档。

【世界观】
设定：现代都市，百年前武道复苏，修炼之人拥有超凡力量，但普通大众不知情
等级体系：练气→筑基→金丹→元婴（四级，不再细分）
社会形态：武道界在暗中运转，有自己的规则和势力，与普通社会平行

【主角设定】
姓名：王峰，30岁，男
表面身份：某公司普通保安，工资3500元，每天站岗
真实实力：金丹巅峰（武道界最顶尖的层次之一），但从未主动暴露
背景：十年前武道界第一天才，经历变故后主动隐居，查明真相前不出手
性格：话不多，看透一切，懒得解释，一旦出手必然碾压，绝不拖泥带水
核心幻想：真正的强者不需要证明自己，实力就是一切

【武道规则——必须精确，后续严格遵守】
筑基：可徒手打碎石头，速度是普通人3倍
金丹：可一拳贯穿30cm厚钢板，百米冲刺不到1秒，可感知气场
普通人无法感知武者气息，除非武者主动释放
王峰当前：金丹巅峰，但刻意压制，外表与普通人无异

【配角设定】
公司老板赵总：有点小钱，看不起保安，经常用钱羞辱下属
前台小美：善良，经常被老板无端刁难，王峰曾默默帮助过她
武道追杀者：当年变故幕后黑手派来的人，开始接近王峰（第7章后出现）

【禁止项】
主角不能主动暴露实力（除非是保护重要的人，或被逼到极限）
武道等级描述必须前后一致
不能出现主角因冲动犯错的情节（他是高手，高手不冲动）
普通人在场时，战斗描写对普通人而言必须看起来很普通

【第一章目标爽点】
公司来了几个"武道高手"（筑基级别），当着所有员工炫耀，专门叫王峰上来当靶子练手。王峰面无表情走上去，对方全力出拳，王峰用两根手指夹住，轻描淡写说了一句话。围观的普通员工懵了，但那几个"高手"瞬间跪地磕头，全场哗然。

输出：结构化设定文档，约600-800字，语言简洁，供后续章节生成直接引用。
```

### Prompt 2：前15章大纲生成（每本书只用一次）

```
基于以下设定，生成【书名】前15章详细大纲。

【设定文档】
{粘贴对应书Prompt 1生成的设定文档}

【大纲格式要求】
每章必须包含以下4项，缺一不可：
1. 章节标题（吸引人，可含悬念感）
2. 本章核心事件（一句话概括）
3. 本章爽点（具体描述读者爽在哪里，不能写"主角变强了"这种废话）
4. 结尾钩子（让读者想看下一章的具体理由）

【整体节奏硬性要求】
第1章：建立主角处境 + 第一个爽点 + 强钩子（最重要的章节）
第2-3章：继续积累反派压迫，展示主角潜力增长
第4-5章：第一次小规模反击，让读者解气
第6-8章：反派升级打压，主角能力持续成长
第9-10章：第一个中型爽点爆发，正面打脸主要反派

【禁止项】
不能有连续2章没有明显爽点
不能有连续1章是纯铺垫无事件发生
每章结尾必须有悬念或期待点，不能平淡收尾

输出格式（严格按照）：
第1章 《章节标题》
核心事件：
本章爽点：
结尾钩子：

第2章 《章节标题》
...
```

### Prompt 3：章节正文生成（每天每章使用）

这是使用频率最高的Prompt。每次使用时替换`{}`中的变量。

```
你是一位专业网文写手，擅长番茄小说男频爽文。请生成第{章节序号}章正文。

━━━━━━━━━ 固定设定（每章必读，严格遵守）━━━━━━━━━
{粘贴对应书的设定文档}

━━━━━━━━━ 本章任务 ━━━━━━━━━
章节标题：{本章标题}
核心事件：{本章核心事件}
本章爽点：{本章爽点描述}
结尾钩子：{本章结尾钩子}

━━━━━━━━━ 上一章结尾（最后200字）━━━━━━━━━
{粘贴上章最后200字。第一章写：无，直接开篇。}

━━━━━━━━━ 写作硬性规则 ━━━━━━━━━

【字数】2000-2200字。少于1900或超过2300均不合格。

【开头规则】
前150字必须有一个具体的冲突或画面，直接进入场景。
禁止以天气描写、环境描写、回忆开头。
禁止第一句是主角起床或回忆往事。

【节奏规则】
每500字必须有一个情节推进点（新信息、新冲突或新进展）。
对话和动作描写交替出现。
禁止超过300字的连续心理独白。
平均句长控制在15字以内，多用短句。

【爽点规则】
本章爽点必须在第1200字之后出现（前面用于积累压抑感）。
爽点出现前，必须让读者替主角憋着一口气。
爽点时主角的反应要冷静克制，不能飘，不能得意忘形。

【结尾规则】
最后100字必须为下一章制造强烈期待。
结尾方式三选一：
- 悬念型：刚发生的事产生了一个未解决的问题
- 冲突型：新的麻烦突然出现
- 高光型：一个让读者想反复看的画面或台词

【禁止词汇和表达】
以下表达一旦出现视为不合格，必须替换：
"此刻他的内心" → 直接写行动
"陷入了沉思" → 直接写他想到了什么
"不知道为什么他感到" → 直接写感受
"就在这时" → 改成具体动作
"突然之间" → 限制使用，全文最多1次
"整个人都" → 删除，直接描写
所有形如"让他感到了XX"的表达 → 直接写"他XX了"

【禁止情节】
不能与设定文档中禁止项的任何内容冲突。
不能与前文已发生的事件矛盾。
不能出现主角来不及思考就做出重大决策的情节。

━━━━━━━━━ 输出要求 ━━━━━━━━━
直接输出正文，不需要任何说明、注解或"以下是正文"之类的前言。
```

### Prompt 4：连续性检查（每5章使用一次）

**在第5章和第10章生成前各运行一次，防止AI忘记前文设定。**

```
请检查以下内容是否存在前后矛盾，并生成当前状态摘要供后续章节使用。

【原始设定】
{粘贴设定文档}

【已发生事件记录】
{按以下格式自己整理，1-2句话/章}
第1章：[关键事件] + [结果] + [主角获得/失去了什么]
第2章：...
第3章：...
第4章：...
第5章：...

【检查项目】
1. 主角能力是否有矛盾？（已获得的能力是否被遗忘或违规使用）
2. 反派状态是否合理？（被打脸后是否仍在不合理地嚣张）
3. 物品/资源数量是否前后一致？
4. 人物关系是否出现无铺垫的突变？
5. 世界规则是否有被违反的地方？

输出格式：

【当前状态摘要】（约150-200字，可直接粘贴到下一章Prompt的"上一章结尾"前使用）
主角当前状态：
已获得能力/资源：
当前主要矛盾：
待解决的悬念：

【发现的矛盾】（如无，填"无"）

【建议注意事项】（给后续章节生成的提醒）
```

### Prompt 5：书名简介优化（必要时使用）

**触发条件：** 发布5天后阅读量仍然低于100，先排除流量问题再考虑内容问题。

```
请为以下网文优化书名和简介，要求符合番茄小说男频爆款风格。

【题材】{题材}
【主角核心设定】{主角能力和爽点方向，2-3句话}
【当前书名】{现有书名}
【当前简介】{现有简介}

【书名要求】
提供3个方向各1个选项：
- 直接暴露金手指型（《开局签到SSS级系统》）
- 身份反差型（《保安王峰》《赘婿陈浩》）
- 结果前置型（《所有人都在等我失败》）

【简介要求】
不超过120字。
结构：弱势起点（1句）→ 金手指出现（1句）→ 冲突期待（1句）→ 主角宣言（1句）
禁止出现："且看主角如何""敬请期待""一场精彩的故事"等废话。
语言要有力，读完想点进去看。

输出：3组 [书名 + 对应简介]
```

## 四、关键节点判断

### 第5天：早期信号检查

| 书架比 | 含义 | 操作 |
|---|---|---|
| > 8% | 内容有吸引力，读者在追 | 继续，不改任何东西 |
| 3% - 8% | 正常水平，需要持续观察 | 继续，重点检查第1章开头是否够强 |
| < 3% 且阅读量 > 200 | 读者进来但不留 | 用Prompt 5优化书名简介 |
| 阅读量 < 50 | 根本没人点进来 | 先换封面和书名，内容是次要问题 |

### 第10天：中期判断

```
任意1本书：
  书架数 > 80，且书架比持续 > 5%  →  有希望，继续写
  书架数 50-80，书架比 3%-5%       →  中等，不急于下结论
  书架数 < 30                       →  这本书大概率失败，记录原因
```

| 结果 | 结论 | 下一步 |
|---|---|---|

## 五、失败诊断

### 类型A：没有流量（阅读量 < 100）

症状：发布后根本没人点进来  
原因：封面/书名/简介没过关，或新书没得到任何推荐  
处理：用Prompt 5重新生成书名简介，换封面，去书荒广场手动推几个帖子  
**不是内容问题，不要改正文。**

### 类型B：有流量但不留人（阅读量 > 200，书架比 < 2%）

症状：读者点进来，很快离开  
原因：第1章开头不够抓人，或AI腔太重  
处理：重点检查第1章前500字，确认有没有大量AI腔，用Prompt 3重新生成第1章并替换  
**这是唯一允许替换已发布章节的情况。**

### 类型C：留存崖（第3-5章后书架增长停滞）

症状：前2章数据还行，之后停滞  
原因：爽点节奏太慢，读者等不及  
处理：在后续章节的Prompt 3中加一条指令："本章爽点必须在1000字内出现，不能超过1500字"  
**不要回去改已发布章节，往后优化。**

### 类型D：系统性失败（3本全部书架比 < 2%）

症状：三个题材全部失败，没有任何正向信号  
处理步骤：
1. 先判断是限流还是质量问题（在读者端搜索自己的书名能否找到）
2. 如果是质量问题，取第1章，用Claude Sonnet重新生成，人工对比质量差异
3. 如果是限流问题，考虑转向七猫小说测试，或等待平台政策调整
