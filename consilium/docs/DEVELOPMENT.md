# 智囊团开发计划

> 版本：v1.1
> 日期：2026-04-05
> 状态：基于评审决策更新

---

## 1. 开发原则

- 客户端优先，服务端极简
- 先跑通一个完整模板，再扩展
- 每个 Phase 结束都有可测试的真实产物
- 讨论引擎和 Prompt 工程是最大时间变量，预留充足缓冲

---

## 2. Phase 1 · 完整版（第 1-12 周）

> **目标**：完整功能内测版本，验证产品体验和稳定性。

### Week 1-2：基础框架

- [ ] Next.js 14 项目初始化，TypeScript 严格模式
- [ ] 目录结构搭建，按 ARCHITECTURE.md 规范
- [ ] CSS 变量主题系统（4 套主题 + `transition: 0.35s`）
- [ ] Zustand store 骨架（template / llm / role / chat / theme）
- [ ] **Vercel AI SDK 集成** + 自定义 Provider 支持
- [ ] IndexedDB 封装（idb 库）
- [ ] 部署到 Vercel，CI/CD 自动部署

### Week 3-4：配置页面

- [ ] 模板选择页：左右布局，模板卡片，分类筛选，主题自动切换，JSON 导入
- [ ] LLM 配置页：5 个子页签，**支持自定义模型（BaseURL + Key + ModelList）**
- [ ] 角色配置页：角色列表，编辑表单，添加/删除角色，主持人 toggle
- [ ] 页面间导航，状态持久化
- [ ] 硬编码三国军师团初始数据

### Week 5-8：讨论引擎（核心，4 周）

**Week 5-6：引擎骨架**
- [ ] Engine 状态机：`EngineState` 定义，phase 转换逻辑
- [ ] `intent.ts`：**LLM 分类**实现用户意图识别
- [ ] `scheduler.ts`：基于意图和 phase 选择下一个发言角色
- [ ] `director.ts`：主持人 prompt 生成，邀请时机判断

**Week 7-8：爽点机制 + 集成**
- [ ] `events.ts`：**全部爽点机制**（打脸/站队/投票/反转）
  - 打脸：关键词冲突检测（简化版）
  - 站队：方向分歧检测
  - 投票：3+ 轮无定论触发
  - 反转：探子来报/叛变/极端假设
- [ ] `rhythm.ts`：发言字数限制注入 prompt，轮次间隔
- [ ] 流式输出接入：消息实时更新，打字指示器
- [ ] 错误处理：超时重试，rate limit 友好提示

### Week 9-10：Prompt 调试 + 引擎完善

- [ ] 三国军师团完整 Prompt 调试：5 个角色各自人设验证
- [ ] 白话文强制约束测试
- [ ] 主持人导演逻辑调试：邀请时机、爽点触发
- [ ] 引擎逻辑根据 Prompt 调试反馈迭代

### Week 11：讨论 UI

- [ ] 聊天界面完整实现：所有消息类型，事件卡，邀请卡，动画
- [ ] 快捷指令动态生成，输入框自适应高度
- [ ] 移动端响应式适配

### Week 12：内测准备

- [ ] E2E 测试：3 个不同话题各跑一遍
- [ ] 找 3-5 个真实用户面对面测试
- [ ] 性能优化：首屏加载、流式渲染性能
- [ ] 错误监控接入（Sentry）
- [ ] 用户数据埋点

## 8. 接手指南

### 8.1 快速上手

```bash
# 1. 克隆仓库，安装依赖
git clone <repo>
npm install

# 2. 配置环境变量
cp .env.example .env.local
# 填入开发用 API Key（至少一个即可）

# 3. 启动开发服务器
npm run dev

# 4. 访问 http://localhost:3000
```

### 8.2 模块边界

| 模块 | 职责 | 不负责 |
|-----|------|------|
| `engine/director.ts` | 决策：调度谁、何时邀请、触发什么事件 | LLM 调用、UI 渲染 |
| `engine/scheduler.ts` | 角色选择算法 | 与 LLM 直接交互 |
| `engine/intent.ts` | 用户意图 LLM 分类 | 调度决策 |
| `llm/client.ts` | 统一 LLM 接口，流式处理，自定义 Provider | 业务逻辑 |
| `store/chat.ts` | 消息列表状态，EngineState 同步 | 引擎逻辑 |
| `components/chat/*` | UI 渲染，只消费 store 数据 | 业务逻辑 |

### 8.3 测试重点

1. 主持人邀请触发时机
2. 打脸机制：关键词冲突场景
3. 用户直接指挥：各种指令格式
4. LLM 异常处理
5. 长对话稳定性（30+ 轮）
